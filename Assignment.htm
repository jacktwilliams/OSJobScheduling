Process Management (Simulation of Time-Shared Multiprogrammed OS)

 

Input:  System and Job Descriptions in the file jobs.dat.

 

The system is described as follows:

 

The first line specifies the timer value.

The second line specifies the degree of multiprogramming.

 

This will be followed by a set of job descriptions - one job per line. The format for the job description is as given below.

 

J P C R C R C R ….. C

 

where:             J is a string – the job name

                        P is an integer – indicating the priority of the job.

                        C is the CPU burst time – an integer.

R will be a character

            I for a page of input, O for a page of output, or T for interactive I/O

 

The jobs.dat file can contain unknown number of job descriptions.

 

Project:            Your program must simulate the state transition diagram, in a time-shared multi-programmed system, for a choice of the policies for job scheduling:

 

a)                  Round-Robin, First Come-First Serve

b)                  Shortest Job (next CPU burst) First

c)                  Priority Scheduling (lower priority number is higher priority)

 

Assumptions:

 

·         Jobs execution starts if it can be included within the degree of multiprogramming.

·         Job execution completes its cpu burst and then does I/O (specified by I, O or T), and then continues its next cpu burst.

·         Jobs will be pre-empted from the CPU if they use up the allocated Time-slice.

·         Time needed for I or O is 50 units of time for every page.

·         Time needed for interactive I/O (T) is 200 units of time.

·         During a context switch all pending interrupts are processed and new requests issued by the OS. That is – queue up all intermediate interrupts/events and process them as part of context switching.

·         Assume all jobs have been submitted at start and there is no overhead in loading the code for the job.

·         There is no buffering and/or spooling.

 

Understand the problem and the sequence of events, then design your software, and then code.

It does 20 units of cpu, then requests a page of output, then does 20 units of cpu, then requests a page of output, then does 50 units of cpu, then does a page of output, and finally does 40 units of cpu.

Output             A chronological tableau of Time and Significant events describing the activities in the system.

 

                        Significant events are

                                    Arrival of job

                                    Scheduling of job

                                    Running of job

                                    Completion of job

                                    Request by job

                                    Input or Output or Interactive i/o started for job

                                    Input or Output or Interactive i/o completed for job

                                    Timer interrupt and timeouts.

                                    Transition of jobs from one state to another.

 

                        Then,

                                    Output the time to complete all jobs

                                    Output for each job the arrival time, completion time, and the turnaround time

                                    Output for each job the amount of time spent in the ready queue.
